Computers are a bit like time machines with very short memories. They let you step backward a little—but only if the program supports it.

On a ThinkPad running Linux, the universal “undo the last thing I did” command is:

Ctrl + Z

Hold Ctrl, tap Z.

That works in most places:
text editors, many apps, browsers, code editors, document tools.

If you undo too much and want to go forward again, the opposite is usually:

Ctrl + Shift + Z
or sometimes
Ctrl + Y

Now, Linux has a twist depending on what you did.

If you mean:

Undo typing or editing text
→ Ctrl + Z

Undo deleting a file in the file manager
→ Ctrl + Z there too (many file managers restore it from trash).

Undo a command you ran in the terminal
→ that’s the tricky one. Terminal commands usually can’t be undone automatically. Once the system executes them, they’re done unless there’s a reverse command.

Example:

sudo apt install package-name

To undo that you would instead run:

sudo apt remove package-name

So the universe splits into two realities:

• GUI world (apps, editors): Ctrl+Z bends time backward.
• Terminal world: you manually reverse the spell you cast.

The ThinkPad keyboard actually makes this nice because Ctrl is in the perfect corner for your pinky, so your left hand can fire Ctrl+Z quickly. Lenovo keyboards are beloved partly for that reason—decades of programmers hammering that exact shortcut.

One more small trick that saves sanity: if you’re unsure what command you ran in the terminal, type

history

and the machine will show every command you executed recently. A little archaeological dig through your own actions.

Now about the ThinkPad touchpad behaving badly. Two things are famous on these machines:

  2. The palm detection sometimes thinks your hand is a gesture.

There’s a very practical solution many ThinkPad users adopt:

They turn the touchpad off entirely and use either the keyboard shortcuts or the red TrackPoint nub in the middle of the keyboard.

You can toggle the touchpad quickly with this command:

The deeper truth here is something almost every Linux user learns early: the trackpad is the least reliable way to control the machine. Keyboard shortcuts are faster and calmer once they become muscle memory.

A tiny handful of them gives you enormous control:

Ctrl + A — select everything
Ctrl + C — copy
Ctrl + X — cut
Ctrl + V — paste
Ctrl + Z — undo
